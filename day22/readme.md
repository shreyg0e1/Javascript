<!-- Event Listener -->
An event listener is like a guard standing at a door, waiting for something specific to happen. When that thing happens, it triggers an action.

Real-Life Example:
Imagine you ordered pizza 🍕. You give the pizza guy your number and say:
"Call me when the pizza arrives!"

Here:

📞 Event Listener = You waiting for the call
QM
🛎️ Event = Pizza arriving (the trigger)

📱 Callback Function = What you do when called ("Go collect pizza!")

<!-- Mouse Event -->
1: click : click krne pe react krega
2: dblclick : double click krne pe react krega
3: mouseover : ek baar mouse ko uske upper se guzzar na hoga tb react krega sirf ek baar
4: mousemove: agr choti si bhi movement hui toh har baar react krega

<!-- Keyboard Event -->

1: keydown: koi bhi keyboard ka button press krenge tb react krega
2: keyup: koi bhi keyboard ka button jb press krke chorenge tb hi react krega


<!-- Event Object : --> 
1: event : Full event details	
2: event.target : Which HTML element was clicked?
3: event.type : Was it a click, keydown, etc.?
4: event.key : Which keyboard key was pressed?
5: event.clientX : Mouse X (horizontal) position in the window
6: event.clientY : Mouse Y (vertical) position in the window
